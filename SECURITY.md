# Security Policy

## Security Notice

Skill code in this repository is curated and maintained, but not independently security-audited.

Use with caution, especially when:
- Running external APIs/services
- Processing sensitive datasets
- Executing scripts in privileged environments

## Reporting a Vulnerability

If you find a security issue, report it privately to the repository maintainer before public disclosure.

Please include:
- Affected file/path
- Reproduction steps
- Impact assessment
- Suggested fix (if available)

## Safe Usage Recommendations

- Review scripts before execution.
- Pin dependencies where practical.
- Run benchmarks and smoke tests in isolated environments.
- Avoid committing sensitive data in `runs/` outputs.
